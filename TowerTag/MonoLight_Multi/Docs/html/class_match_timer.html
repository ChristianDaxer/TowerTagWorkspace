Detailed Description

Class to make central MatchTimer available for all classes who need the Match/Countdown-Timer.

Member Enumeration Documentation

◆ TimerState

enum MatchTimer.TimerState
strong

TimerState:

  • Undefined: We are not in a Match (before MatchStartsAt and after StopMatch is called).
  • MatchTimer: Match is running.
  • CountdownTimer: Countdown after MatchStartsAt or ResumeMatch was started (when WaitingForCountdown phase is over).
  • TimerIsPaused: When Match is paused (after PauseMatch until ResumeMatch is called).
  • WaitForCountdown: Time before the countdown begins (MatchStartsAt or ResumeMatch was called and before the countdwon is started).

◆ GetCountdownTime()

int MatchTimer.GetCountdownTime ( )

Calculates and returns the current countdown timespan (timespan from current timestamp to the start of the match (if set appropriate by StartTimerAt(..) or ResumeTimerAt(..))).

If Timer is not in countdown state (TimerState.CountdownTimer != currentTimerState) it returns the countdown time set by last StartTimerAt(..) or ResumeTimerAt() call.

Returns
If Timer is in CountdownTimer state (TimerState.CountdownTimer == currentTimerState) it returns Timespan from current timestamp to MatchEndsAt timestamp in seconds [countdownTime..0], otherwise the last set countdown.

◆ GetCurrentTimerInSeconds()

float MatchTimer.GetCurrentTimerInSeconds ( )

Convinience function to calculate Timer values dependent of the internal state.

Use this in conjunction with isTimerValid or currentTimerState to check if the returned values are valid and what they mean.

Returns
Timespan to the end of the match when in MatchTimerState (same as GetRemainingMatchTimeInSeconds()). Timespan to the start of the match when in CountdownTimerState (same as GetCountdownTime()). CountdownTime in seconds when in WaitForCountdownState (same as countdownTimeInSeconds). -1 when in another TimerState.

void MatchTimer.ResumeTimerAt ( int  resumeTimestamp,
int  newEndTimestamp,
int  countdownTimeInSeconds 
)

Resume Timer from Pause.

To calculate new EndTimestamp call GetTimeTillEndOfMatch() before resuming the Timer:

  • startTimestamp = currentTiemstamp + countdownTime + offsetToCompensateNWLatency
  • endTimestamp = startTimestamp + GetRemainingMatchTimeInSeconds()
Parameters
resumeTimestampTimestamp the match should resume from pause.
newEndTimestampTimestamp the match should end.
countdownTimeInSecondsCountdown to show before resuming from pause (if it is enough time till resumeTimestamp).

Starts timer and sets timestamps needed for timer calculations.

Please calculate appropriate timestamps beforehand:

  • startTimestamp = currentTiemstamp + countdownTime + offsetToCompensateNWLatency
  • endTimestamp = startTimestamp + matchTime
Parameters

endTimestampTimestamp the timer switches to undefined state (when the match should end).
